How it works

Every morning a GitHub Actions job:

  1. Pulls the latest results from the official MLB Stats API (2015–2026, ~26,000+ games).
  2. Scores yesterday's predictions and appends them to a permanent log — predictions are always made before games are played, so the Model Tracker is a true out-of-sample track record.
  3. Rebuilds each team's season-to-date profile from per-game logs — win/loss form, run differential, and advanced stats (OPS, ERA, FIP, WHIP, Pythagorean expected win%) — then retrains three win-probability models on the expanded data: a FiveThirtyEight-style Elo baseline, logistic regression, and XGBoost. All features are computed strictly as-of first pitch (no leakage).
  4. Runs 10,000 Monte Carlo simulations of the remaining schedule and the full 12-team postseason bracket to produce playoff, pennant, and World Series odds.
  5. Fetches injury-list moves and call-ups, and publishes everything to this page.

Single MLB games are close to coin flips — the best teams still lose 60+ times a year — so the honest benchmark is log loss and calibration against the Elo baseline, not raw accuracy. Starting pitchers and injuries are not yet model features; they are the next planned iteration.
